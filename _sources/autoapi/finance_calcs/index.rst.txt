finance_calcs
=============

.. py:module:: finance_calcs

.. autoapi-nested-parse::

   finance-calcs — financial calculations as polars expressions.

   Every public function accepts and returns ``pl.Expr``. Functions are
   also exposed on a ``.finance`` custom namespace bound to ``pl.Expr``
   and ``pl.Series``::

       import finance_calcs  # noqa: F401  — registers the namespace
       import polars as pl

       df.with_columns(
           pl.col("close").finance.log_returns().alias("ret"),
       ).select(
           pl.col("ret").finance.sharpe().alias("sharpe"),
           pl.col("ret").finance.max_drawdown().alias("mdd"),
       )

.. py:function:: slippage_bps(execution_price: polars.Expr, benchmark_price: polars.Expr, *, side: polars.Expr | None = None) -> polars.Expr

   Execution slippage in basis points.

   Without ``side``, the result is signed price difference versus the
   benchmark. With ``side``, positive values mean adverse execution cost
   for buy/cover and sell/short transactions.


.. py:function:: transaction_cost(quantity: polars.Expr, price: polars.Expr, *, commission: float | polars.Expr = 0.0, fees: float | polars.Expr = 0.0, bps: float | polars.Expr = 0.0) -> polars.Expr

   Per-trade cost from explicit charges plus basis-point slippage.

   ``bps`` is applied to absolute traded notional. ``commission`` and
   ``fees`` may be scalars or expressions aligned to the transaction rows.


.. py:function:: transaction_notional(quantity: polars.Expr, price: polars.Expr) -> polars.Expr

.. py:function:: turnover(weights: polars.Expr, *, window: int | None = None) -> polars.Expr

   Portfolio turnover contribution from position-weight changes.

   Apply over a symbol/security partition, then aggregate by rebalance
   date. The contribution is ``0.5 * abs(weight - prior_weight)``.

.. py:function:: long_short_spread(returns: polars.Expr, quantile: polars.Expr, upper: int, lower: int) -> polars.Expr

   Top-quantile mean return minus bottom-quantile mean return.

   Use inside ``group_by("date").agg(...)``::

       df.group_by("date").agg(
           long_short_spread(pl.col("ret"), pl.col("q"), upper=4, lower=0)
           .alias("ls"),
       )

   :param returns: Forward return series.
   :param quantile: Integer quantile label series.
   :param upper: Long quantile label.

.. py:function:: gpd_cvar(returns: polars.Series, var_p: float = 0.01, threshold_p: float = 0.1) -> float

   GPD-fitted extreme CVaR (expected shortfall beyond ``var_p``).

   Closed form for the GPD tail (xi < 1):
       :math:`CVaR_p = \frac{VaR_p}{1 - \xi} + \frac{\beta - \xi u}{1 - \xi}`

   :param returns: Periodic returns.
   :param var_p: Tail probability.
   :param threshold_p: Mass beyond the threshold used for the fit.

   :returns: CVaR magnitude as a positive float.


.. py:function:: gpd_var(returns: polars.Series, var_p: float = 0.01, threshold_p: float = 0.1) -> float

   GPD-fitted extreme VaR (positive number, magnitude of loss).

   Fits a Generalized Pareto Distribution to the excess of losses
   over a threshold (peak-over-threshold) and inverts to obtain the
   ``var_p`` quantile.

   Closed form:
       :math:`VaR_p = u + \frac{\beta}{\xi}\left(\left(\frac{n}{n_u} p\right)^{-\xi} - 1\right)`

   :param returns: Periodic returns (``pl.Series``).
   :param var_p: Tail probability (``0.01`` → 1% VaR).
   :param threshold_p: Probability mass beyond the threshold ``u``
                       used for the GPD fit (``0.10`` → top-10% of losses).

   :returns: VaR magnitude as a positive float.
